# 项目骨架规划

本文档描述宠物急救分诊与到院前协同平台未来工程阶段的技术骨架规划。

> ⚠️ 当前处于规划文档阶段，以下目录结构和技术方案尚未实施。

---

## 一、技术选型

| 层次 | 技术方案 | 说明 |
|------|----------|------|
| 用户端 | Taro + React + TypeScript | 编译为微信小程序 |
| 医院后台 | Next.js + TypeScript | Web 管理界面 |
| 后端 API | NestJS + TypeScript | RESTful API 服务 |
| 数据库 ORM | Prisma | 类型安全的数据库访问 |
| 数据库 | PostgreSQL | 主数据库 |
| 实时推送 | WebSocket / 微信订阅消息 | SOS 到院提醒 |
| 文件存储 | 对象存储（OSS/COS） | 照片等附件 |
| 部署 | 容器化（Docker） | 后端和后台部署 |

## 三、核心数据模型（规划）

### 3.1 SOS 急救记录（SosRecord）

```typescript
// 规划中的数据结构，尚未用 Prisma 实现
interface SosRecord {
  id: string;
  createdAt: Date;
  status: 'pending' | 'in_progress' | 'resolved' | 'closed';

  // 用户信息（脱敏后存储）
  userId: string;
  contactPhone: string;

  // 位置信息
  location: {
    latitude: number;
    longitude: number;
    address: string;
  };

  // 宠物信息
  petId: string;
  petName: string;
  petSpecies: string;
  petBreed: string;
  petAge: number;

  // 症状信息（用户填写）
  symptomSummary: string;
  symptomDuration: string;
  consciousnessStatus: 'conscious' | 'unconscious' | 'unknown';
  hasBleedingOrTrauma: boolean;

  // 后台标记（用户不可见）
  redFlag: boolean;
  redFlagReason?: string;

  // 目标医院
  targetHospitalId: string;

  // 处理记录
  handledBy?: string;       // 医生 ID
  handledAt?: Date;
  callbackAt?: Date;
  doctorNotes?: string;
  caseSummary?: string;
}
```

## 四、核心 API 端点（规划）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/sos` | 提交 SOS 请求 |
| GET | `/sos/:id` | 获取 SOS 详情 |
| PATCH | `/sos/:id/status` | 更新处理状态（医院端） |

| GET | `/hospitals/nearby` | 获取附近医院列表 |
| GET | `/pets` | 获取用户宠物档案列表 |
| GET | `/pets/:id` | 获取宠物详情 |

## 五、SOS 流程页面规划（小程序）

### 5.1 页面列表

| 页面路径 | 功能 |
|----------|------|
| `pages/index/index` | 首页，核心 SOS 大按钮 |
| `pages/sos/step1` | 症状选择（快选 + 文字输入） |
| `pages/sos/step2` | 确认自动补全信息 |
| `pages/sos/step3` | 选择目标医院 |
| `pages/sos/confirm` | 确认提交 |
| `pages/sos/success` | 提交成功，等待回拨 |
| `pages/history/index` | 历史 SOS 记录 |
| `pages/profile/index` | 宠物档案入口 |

### 5.2 首页设计原则

- SOS 按钮占据屏幕视觉中心，不可遮挡
- 按钮颜色：紧急红色（#E53935）
- 按钮文字：「🆘 立即求助」
- 点击后立即进入 SOS 流程，不弹窗确认（减少操作步骤）

## 六、医院后台功能规划

| 功能模块 | 说明 |
|----------|------|
| SOS 待处理列表 | 实时显示新 SOS，支持红旗优先排序 |
| SOS 详情 | 完整信息展示，含红旗标记和症状摘要 |
| 电话记录 | 记录回拨时间和医生指导内容 |
| 病例摘要 | 自动生成 + 医生手动核准 |
| 历史归档 | 按状态、时间、医生筛选历史记录 |
| 统计报表 | SOS 量、响应时间、处理率等指标 |

## 七、进入工程阶段的前置确认清单

在正式初始化工程前，项目所有者需确认以下事项：

- [ ] 确认采用 Taro + React + TypeScript 作为小程序框架
- [ ] 确认采用 NestJS + Prisma 作为后端方案
- [ ] 确认采用 Next.js 作为医院后台方案
- [ ] 确认 Monorepo 管理工具（pnpm workspaces / Turborepo）
- [ ] 确认云服务商（阿里云 / 腾讯云）
- [ ] 确认微信小程序 AppID
- [ ] 确认首批合作医院名单和对接方式
- [ ] 确认数据库部署方案（云数据库 / 自托管）
- [ ] 确认用户隐私政策和合规要求
