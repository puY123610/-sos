# 分支与回滚

## 基本原则

`main` 必须保持稳定。

以下工作开始前必须创建专门分支：

- 工程实现。
- 脚手架搭建。
- 依赖安装。
- 大型文档迁移。
- 数据库模型调整。
- 医疗安全规则调整。

## 推荐分支流程

```bash
git switch main
git pull
git switch -c feature/<short-task-name>
```

示例：

```bash
git switch -c feature/wechat-miniapp-skeleton
git switch -c feature/sos-flow-docs
git switch -c feature/hospital-admin-skeleton
git switch -c feature/chinese-doc-structure
```

## Review 规则

- Owner 审阅后再决定是否合并。
- 不满意时不合并 feature 分支。
- 满意后再合并回 `main` 并推送。
- 不直接在 `main` 上做实验性脚手架或大功能。

## 未提交修改的回滚

如果文件已经修改但未提交：

```bash
git restore .
git clean -fd
```

注意：`git clean -fd` 会删除未跟踪文件，执行前必须确认这些文件不需要保留。

## 丢弃未合并分支

如果工作都在 feature 分支，且 Owner 决定不要这次结果：

```bash
git switch main
git branch -D feature/<short-task-name>
```

## 已提交或已推送后的回滚

如果提交已经共享，优先使用：

```bash
git revert <commit-sha>
```

不要擅自使用：

```bash
git reset --hard
```

除非 Owner 明确批准并理解风险。

## 提交前检查

提交前至少检查：

```bash
git status --short
```

如果项目已有脚本，再运行：

```bash
pnpm lint
pnpm typecheck
pnpm test
pnpm build
```

当前工程骨架阶段还没有这些脚本，不能伪装成已经通过。
