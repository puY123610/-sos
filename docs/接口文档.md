# 接口文档

## 当前状态

本文件只记录 planned API。当前仓库没有 NestJS 工程、控制器、数据库模型或业务实现。

以下接口是后续实现方向，不代表现在已经可调用。

## 通用约定

- API 由 `services/api` 提供。
- 前端通过 `packages/api-client` 访问后端。
- 前后端共享类型放在 `packages/shared-types`。
- 每条 SOS 记录必须带 `schemaVersion`。
- 权限、医疗安全和脱敏规则必须由后端执行，不能只依赖前端隐藏。

## 用户端 planned API

### 创建 SOS 案例

```text
POST /v1/sos-cases
```

用途：

- 接收用户 SOS 极简表单。
- 记录联系方式、定位快照、宠物档案 ID、提交时间和表单版本。
- 进入医院处理队列。
- 后端生成后台红旗标记，但不返回给用户端展示为风险等级。

核心字段方向：

- 宠物类型/品种。
- 当前状态。
- 血型，可为未知。
- 用户联系方式。
- 微信用户标识。
- 定位或手动选择位置。
- 宠物档案 ID。
- 备注。
- `schemaVersion`。

用户端响应方向：

- 提交是否成功。
- 案例 ID。
- 已发送医院的提示。
- 保持电话畅通的提示。

不得返回：

- 诊断结果。
- 用药建议。
- 药物剂量。
- 红黄绿风险等级。
- “无需去医院”或“在家处理即可”等话术。

### 查询附近医院

```text
GET /v1/hospitals/nearby
```

用途：

- 根据定位或手动选择位置返回可联系医院。
- 支持距离排序。
- 后续接入高德地图供应商适配。

注意：

- 真实医院电话、地址、接诊状态必须有数据来源和校验记录。
- 未验证医院资料不得用于线上承诺。

### 宠物档案

```text
GET /v1/pet-profiles
POST /v1/pet-profiles
PATCH /v1/pet-profiles/{petProfileId}
```

用途：

- 管理宠物基础资料。
- 为 SOS 提交补充结构化信息。

不得提交真实用户数据到仓库。

## 医院后台 planned API

### 查询 SOS 队列

```text
GET /v1/hospital/sos-cases
```

用途：

- 医院人员查看分配给本医院的 SOS 队列。
- 后台可以展示红旗标记用于优先处理。

权限要求：

- 医院只能访问分配给自己的案例。
- 不得只依赖前端隐藏权限。

### 查询 SOS 详情

```text
GET /v1/hospital/sos-cases/{sosCaseId}
```

用途：

- 查看宠物类型、当前状态、血型、联系方式、定位、提交时间、后台红旗标记和案例状态。

### 记录医院处理结果

```text
POST /v1/hospital/sos-cases/{sosCaseId}/handling
```

用途：

- 记录是否接通。
- 记录医生备注。
- 标记是否建议到院。
- 关闭案例。

医生备注属于敏感业务数据，后续脱敏导出时必须处理。

## 运营后台 planned API

### 医院资料维护

```text
GET /v1/admin/hospitals
POST /v1/admin/hospitals
PATCH /v1/admin/hospitals/{hospitalId}
```

用途：

- 维护医院资料。
- 记录医院数据来源。
- 记录人工校验状态。

### 脱敏训练样本导出

```text
POST /v1/admin/ai-training-samples/export
```

用途：

- 从授权数据中生成脱敏训练样本。
- 仅用于后续 AI 摘要、优先级辅助和数据分析。

必须移除：

- 手机号。
- 姓名。
- 微信 openid。
- 精确地址。
- 精确经纬度。
- 医院私密备注。
- 其他可识别个人身份的信息。

## AI planned API

AI 能力 MVP 默认关闭。

后续可预留：

```text
POST /v1/internal/ai/case-summary
POST /v1/internal/ai/priority-assist
```

限制：

- 只允许内部后端调用。
- 默认受 `AI_FEATURE_ENABLED=false` 控制。
- 不得参与用户端诊断或风险降级。
- 不得开药、给剂量或承诺安全。

## 后续实现前置条件

实现任何 API 前必须先明确：

- 鉴权方案。
- 租户和医院隔离方案。
- Prisma 数据模型。
- 日志和审计字段。
- 测试覆盖范围。
- 医疗安全测试场景。
