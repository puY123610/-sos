# 宠物急救分诊与到院前协同平台

这是一个宠物急救分诊与到院前协同平台的规划仓库。

当前仓库不是微信开发者工具项目，也不是已经初始化好的前端或后端工程。当前阶段只保存项目规则、产品规划、SOS 急救流程、未来工程骨架和交付物。

## 项目定位

本项目不是 AI 宠物医生。

MVP 目标是帮助宠物主人在紧急情况下快速完成：

1. 点击首页大 SOS 按钮。
2. 回答尽量少的关键问题。
3. 系统自动补充联系方式、定位、宠物档案和提交时间。
4. 将 SOS 信息发送到医院后台。
5. 医生通过电话指导紧急措施。
6. 平台沉淀结构化、可脱敏的数据，为后续 AI 辅助能力预留基础。

用户端不展示红黄绿风险等级；后台可保留红旗标记，用于医院优先处理和安全兜底。

## 医疗安全边界

系统不得：

- 诊断疾病
- 开药
- 提供药物剂量
- 承诺安全或治愈
- 建议红旗症状下延迟就医
- 让 AI 替代医生指导急救

系统可以：

- 收集结构化急救信息
- 将 SOS 信息发送给医院后台
- 支持医生回拨和处理记录
- 生成病例摘要
- 在后台保留红旗标记
- 为后续授权、脱敏后的 AI 训练沉淀数据

## 后续进入工程阶段的前置条件

进入工程阶段前，需要明确：

- 是否开始初始化微信小程序工程
- 是否采用 Taro + React + TypeScript
- 是否同时初始化医院后台和后端 API
- 是否需要先创建 GitHub/Gitee 远程仓库
- 首批真实合作医院数据来源和验证方式
