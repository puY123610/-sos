# 用户端小程序骨架

用户端小程序骨架。

## 定位

- MVP 首发端。
- 第一屏强调大 SOS 急救按钮。
- 用户点击 SOS 后只回答最少问题。
- 提交后提示信息已发送给医院，并提醒保持电话畅通。

## 技术方向

正式开发时采用 Taro + React + TypeScript。

当前没有初始化 Taro 工程，没有安装依赖，也没有页面实现代码。

## 后续 App 化路径

- 小程序优先沉淀移动端页面结构、接口契约和业务流程。
- H5 可沿用 Taro 多端能力。
- 原生 iOS / Android 或 React Native App 需要后续单独评审，不在 MVP 当前骨架中创建。
- 未来 App 应复用 `packages/shared-types`、`packages/api-client` 和后端 API 契约，避免重写业务规则。

## 规划页面

- 首页 SOS
- SOS 极简表单
- 定位授权或手动选址
- 医院匹配
- 提交成功
- 宠物档案

用户端不得展示红黄绿风险等级，不得提供诊断、用药、剂量或安全承诺。
